# Chrome 进程架构


我们先了解下支撑着 Chorme 浏览器的进程架构，从上图图中可以看出，最新的 Chrome 浏览器包括：
浏览器(Browser)主进程、网络(NetWork)进程、 渲染进程、GPU进程和插件进程。

# 1、浏览器进程

主要负责用户交互、界面显示、子进程管理，同时提供存储等功能。

# 2、网络进程

主要负责页面的网络资源加载，如用户输入网址后请求资源。

# 3、渲染进程

网络进程请求回来资源后，将 HTML、CSS 和 JavaScript 转换为用户可以与之交互的网页，排版引擎 Blink 和 JavaScript 引擎V8都是运行在该进程中，默认情况下，Chrome 会为每个 Tab 标签创建一个渲染进程（打开的新页面和当前页面为同一站点时，新页面会使用父页面的渲染进程）。渲染进程都是运行在沙箱模式下。

# 4、GPU进程

渲染流畅的3D动画、普通页面，提高浏览器绘制的效率。Chrome 的 GPU 进程的背后

# 5、插件进程

主要是负责插件的运行，因插件易崩溃，所以需要通过插件进程来隔离，以保证插件进程崩溃不会对浏览器和页面造成影响。
